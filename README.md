# Производственный Telegram-бот

Простой бот для учёта производства, склада, сырья, счётчиков, сборки, отгрузки и файлов для печати.

## Запуск

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Создайте файл `.env` рядом с `.env.example`.

3. Вставьте новый API-ключ бота только в `.env`:

```env
BOT_TOKEN=PASTE_NEW_TOKEN_HERE
GLOBAL_OWNER_IDS=2097006037
BOT_DATA_DIR=./data
```

4. Перед запуском проверьте настройки:

```bash
python scripts/live_start_check.py
```

5. Запустите:

```bash
python -m app.main
```

На Windows можно открыть `start_windows.bat`.
На Linux можно выполнить `./start_linux.sh`.

## Важно

- не отправляйте API-ключ в чат;
- не добавляйте `.env` в архивы и репозитории;
- если API-ключ был показан посторонним, перевыпустите его в BotFather;
- бот берёт ключ только из `.env`;
- в архиве должен быть только `.env.example`.

## Работа

Бот запускается пустым. Названия участков, изделий, комплектующих, сырья, складских позиций, счётчиков и должностей создаются пользователем.

Подсказки показываются прямо во время настройки: нажмите кнопку, и бот скажет, что ввести дальше.

## Ввод данных

Можно писать коротко:

```text
Сделали Деталь 1 300
Расход Сырьё 1 25 кг
Привезли Сырьё 1 500 кг
Счётчик 1 1356.7
Собрано Изделие 1 20
Отправлено Изделие 1 10
```

Можно писать одним сообщением:

```text
Производство Участок 1
Деталь 1 300
Деталь 2 500
Расход Сырьё 1 25 кг
Счётчик 1 1356.7
```

Перед сохранением бот показывает сводку и ждёт подтверждение кнопкой или текстом `да`, `ок`, `верно`.

## Отчёты

Можно запросить один день или период:

```text
отчёт 12.05.2022
отчёт 12.05.2022-20.06.2022
```

Если дата начала позже даты конца, бот попросит исправить период.

## Файлы

Доступны форматы:

- Excel;
- PDF;
- CSV;
- HTML;
- TXT;
- ZIP со всеми форматами.
